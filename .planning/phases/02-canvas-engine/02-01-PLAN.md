    Define ALL engine types. This is the contract for the entire Canvas engine:
    - `TileType` enum: `VOID = 0, FLOOR = 1, WALL = 2, DOOR = 3` (numeric for fast tile map lookups)
    - `Direction`: `'up' | 'down' | 'left' | 'right'`
    - `CharacterState`: `'idle' | 'walk' | 'work'`
    - `TileCoord`: `{ col: number; row: number }`
    - `Room`: `{ id: string; name: string; tileRect: { col: number; row: number; width: number; height: number }; doorTile: TileCoord; seatTile: TileCoord; billyStandTile: TileCoord }`
    - `Character`: `{ id: string; x: number; y: number; tileCol: number; tileRow: number; state: CharacterState; direction: Direction; frame: number; frameTimer: number; path: TileCoord[]; moveProgress: number; speed: number }`
    - `Camera`: `{ x: number; y: number; zoom: number; targetX: number; targetY: number; followTarget: string | null }`
    - `SpriteFrame`: `{ x: number; y: number; w: number; h: number }`
    - `TILE_SIZE = 16` constant (per research: 16x16 tiles, displayed at 2x zoom as 32x32)
    - `WALK_SPEED = 64` (pixels/sec base), `WALK_SPEED_FAST = 128` (long distance), `SPEED_RAMP_TILES = 8` (threshold)
    - `WALK_FRAME_DURATION = 0.15` (seconds per walk animation frame)
    - `WORK_FRAME_DURATION = 0.5` (seconds per work animation frame)

    **Step 3: Create `src/engine/tileMap.ts`.**
    Pure functions operating on 2D tile arrays:
    - `isWalkable(col, row, tileMap: TileType[][])`: bounds check + tile is FLOOR or DOOR
    - `findPath(startCol, startRow, endCol, endRow, tileMap)`: BFS on 4-connected grid (up/down/left/right). Returns `TileCoord[]` from start to end (excluding start). Returns empty array if no path found. Use `Set<string>` for visited with key `"col,row"`, `Map<string,string>` for parent reconstruction.
    - `getTileAt(col, row, tileMap)`: returns TileType at position, or VOID if out of bounds
    - `createTileMap(width, height, fill: TileType)`: creates a 2D array filled with the given type

    **Step 1: Create `src/engine/camera.ts`.**
    Camera system with integer zoom:
    - `createCamera()`: returns Camera with default zoom=2 (follow mode), x/y at center of BILLY's starting room
    - `updateCamera(camera, dt, canvasWidth, canvasHeight)`: smooth lerp toward targetX/targetY. `camera.x = camera.x + (camera.targetX - camera.x) * 0.1`. Use Math.round() on final camera.x/y to keep pixel-perfect alignment.
    - `screenToTile(screenX, screenY, camera, canvasWidth, canvasHeight)`: converts CSS click coordinates to tile col/row. Accounts for zoom, camera offset, and map centering. Returns `TileCoord | null` (null if outside map bounds).
    - `tileToScreen(col, row, camera, canvasWidth, canvasHeight)`: converts tile position to screen pixel coordinates for overlays.
    - Zoom levels: 1 = overview (whole floor visible), 2 = follow (camera tracks target). Integer only, enforced by Math.round().

    **Step 2: Create `src/engine/spriteSheet.ts`.**
    Sprite loading and caching system:
    - `loadSpriteSheet(src: string): Promise<HTMLImageElement>`: loads an image, returns promise that resolves on load
    - `loadAllAssets(): Promise<void>`: loads all required sprite sheets. For Phase 2, this is a no-op or loads minimal placeholder images. The function exists as the integration point for Phase 8 production sprites.
    - `getCachedSprite(sheet, frame, zoom): HTMLCanvasElement`: extracts a frame from a sprite sheet, caches a pre-scaled canvas per zoom level. Sets `imageSmoothingEnabled = false` on the cached canvas context.
    - `clearSpriteCache()`: clears all cached sprites (called on zoom change)

    For Phase 2 placeholder rendering: Instead of sprite sheets, the renderer will draw colored rectangles. The spriteSheet module provides the caching infrastructure that Phase 8 will use for real assets. Export a `PLACEHOLDER_COLORS` map: `{ floor: '#3a3530', wall: '#1a1815', door: '#4a4540', billy: '#e8a838', diana: '#c084fc', marcos: '#60a5fa', sasha: '#34d399', roberto: '#f87171', valentina: '#fb923c', 'war-room-floor': '#2a2d35' }` using the project's warm/cool palette.

    **Step 3: Create `src/engine/renderer.ts`.**
    Layered rendering pipeline:
    - `renderFrame(ctx, state, canvasWidth, canvasHeight)`: main render function called every frame.
      - Layer 1: Clear canvas with background color (#0d0b09, matching app dark theme)
      - Layer 2: Floor tiles -- iterate visible tiles only (camera viewport culling). Draw colored rectangles for each TileType: FLOOR gets warm color for offices (detect via getRoomAtTile) or cool for hallways, WALL gets dark color, DOOR gets slightly lighter floor color. Skip VOID tiles.
      - Layer 3: Furniture placeholders -- draw simple rectangles for desks (brown), chairs (gray), conference table (dark wood). Position data comes from officeLayout furniture array.
      - Layer 4: Characters -- draw colored rectangles with a directional indicator (small triangle or dot). Sort by Y position for depth. Size: TILE_SIZE pixels.
      - Layer 5: UI overlays -- room name labels when BILLY is in a room, room highlight on hover/selection, BILLY position marker in overview zoom.
    - All positions use integer math: `Math.floor()` or `Math.round()` for pixel-perfect rendering.
    - Set `ctx.imageSmoothingEnabled = false` at start of every frame.
    - Viewport culling: only draw tiles where `col * tileSize * zoom + offsetX` is within `[0, canvasWidth]` range.

    **Step 4: Create `src/engine/gameLoop.ts`.**
    Game loop independent of React:
    - `startGameLoop(canvas: HTMLCanvasElement): () => void`
    - Uses `requestAnimationFrame` for 60fps timing
    - Delta time calculation with cap: `Math.min((time - lastTime) / 1000, 0.1)` to prevent jumps on tab switch
    - Each frame: read officeStore via `useOfficeStore.getState()` (non-reactive), call `updateCharacters(dt)`, call `updateCamera(dt)`, call `renderFrame(ctx, ...)`
    - Returns cleanup function that sets `stopped = true` and calls `cancelAnimationFrame(rafId)`
    - HiDPI setup happens OUTSIDE the game loop (in the React mount effect). The game loop receives an already-configured canvas.
    - Canvas resize handling: check `canvas.getBoundingClientRect()` dimensions against stored values. If changed, re-run HiDPI setup.

    **Step 5: Expand `src/store/officeStore.ts`.**
    Replace the skeleton store with full state. Keep the same flat Zustand pattern as chatStore:
    ```typescript
    interface OfficeState {
      // Room data
      rooms: Room[];
      activeRoomId: string | null;
      targetRoomId: string | null;

      // BILLY
      billyTileCol: number;
      billyTileRow: number;

      // Camera
      zoomLevel: number; // 1 = overview, 2 = follow

      // Characters (engine manages these, store holds the source of truth)
      characters: Character[];

      // Actions (only called on meaningful state changes, NOT every frame)
      setActiveRoom: (roomId: string | null) => void;
      setTargetRoom: (roomId: string | null) => void;
      setBillyPosition: (col: number, row: number) => void;
      setZoomLevel: (level: number) => void;
      initializeCharacters: () => void;
    }
    ```
    Initialize rooms from `ROOMS`, billyTileCol/Row from BILLY's room seatTile, zoomLevel=2.
    `initializeCharacters()` creates Character objects for BILLY and 5 agents at their seatTile positions with state='idle'.

    - Test viewport culling: tiles outside viewport are not drawn (mock ctx.fillRect, verify call count)
    - Test zoom=1 and zoom=2 produce different tile sizes in draw calls
