    - Test: Character pixel position (x, y) interpolates between current tile and next tile based on moveProgress
    - Test: When BILLY arrives at an agent room, a brief 'knock' pause occurs (0.5s idle before entering)
    - Test: Agents in 'work' state transition to 'idle' and face BILLY when BILLY enters their room

  <action>
    Create `src/engine/characters.ts` with character state machine logic:

    **`updateCharacter(ch: Character, dt: number, tileMap: TileType[][]): void`**
    Main update function. Switch on ch.state:
    - `'walk'`: Advance moveProgress by `(ch.speed / TILE_SIZE) * dt`. When moveProgress >= 1, snap to next tile (`ch.path.shift()`), update tileCol/tileRow, reset moveProgress to overflow amount. Calculate direction from movement delta. Cycle walk animation frame. Interpolate x/y pixel position between current and next tile using moveProgress. If path exhausted, transition to 'idle'.
    - `'work'`: Cycle animation frame every WORK_FRAME_DURATION. No position change.
    - `'idle'`: No frame cycling, no position change.

    **`updateAllCharacters(dt: number, tileMap: TileType[][]): void`**
    Read characters from `useOfficeStore.getState()`. Call updateCharacter for each. After updating BILLY:
    - Check if BILLY just arrived at a new tile (tileCol/tileRow changed)
    - Use `getRoomAtTile` to detect room entry
    - If BILLY entered an agent's room: call `useOfficeStore.getState().setActiveRoom(room.id)`, set `useOfficeStore.getState().setBillyPosition(col, row)`
    - Handle knock animation: when BILLY's path ends at a room's billyStandTile, insert a 0.5-second pause (set a `knockTimer` on the character) before updating activeRoom. During knock pause, character is 'idle' at the door tile, then walks to billyStandTile.
    - When BILLY enters a room, find the agent character in that room and transition them from 'work' to 'idle', setting their direction to face BILLY (compute direction from agent's position to BILLY's position).

    **`startWalk(characterId: string, targetCol: number, targetRow: number, tileMap: TileType[][]): void`**
    Finds character in store, computes BFS path via `findPath`, assigns path to character, sets state to 'walk', calculates speed (WALK_SPEED if path.length <= SPEED_RAMP_TILES, else WALK_SPEED_FAST), resets moveProgress to 0.

    **`getCharacterDirection(fromCol: number, fromRow: number, toCol: number, toRow: number): Direction`**
    Returns the Direction based on delta between two tile positions. Prioritize vertical if both axes change.

    Write `src/engine/__tests__/characters.test.ts` with all tests from the behavior block. Mock `useOfficeStore.getState()` to return test character data. Mock `findPath` where needed.

    **`setupInputHandlers(canvas: HTMLCanvasElement): () => void`**
    Attaches click event listener to canvas. On click:
    1. Get CSS coordinates relative to canvas via `getBoundingClientRect()`
    2. Read camera state from `useOfficeStore.getState()`
    3. Convert to tile coordinates via `screenToTile(cssX, cssY, camera, canvasWidth, canvasHeight)`
    4. If tile is valid, check `getRoomAtTile(col, row)` to find target room
    5. If room found and room.id !== current activeRoomId:
       - Call `useOfficeStore.getState().setTargetRoom(room.id)`
       - Call `startWalk('billy', room.billyStandTile.col, room.billyStandTile.row, OFFICE_TILE_MAP)` to begin BILLY's walk
    6. If click is on hallway (no room), ignore (BILLY only walks to rooms, per user decision)
    Returns cleanup function that removes the event listener.

    Also handle zoom toggle: Listen for keyboard 'z' key or a double-click to toggle between zoom levels 1 and 2. Call `useOfficeStore.getState().setZoomLevel(newLevel)`.

    Also track mouse position for room hover highlighting (store in a module-level variable, read by renderer for drawing hover highlight).
