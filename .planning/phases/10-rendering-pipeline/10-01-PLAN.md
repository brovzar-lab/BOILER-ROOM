<objective>
Refactor the rendering pipeline from manual `* zoom` coordinate multiplication to `ctx.setTransform(zoom, 0, 0, zoom, offsetX, offsetY)` and make the entire zoom system float-compatible.

Purpose: Eliminates tile gaps at fractional zoom (RNDR-04) by letting the canvas transform handle sub-pixel math uniformly. Makes zoom a float throughout the pipeline (RNDR-01), which is the structural prerequisite for Phase 12's smooth pinch-to-zoom. Quantizes sprite cache keys to prevent memory explosion at fractional zoom values.

Output: renderer.ts draws world content via setTransform; camera.ts supports float zoom with no Math.round snapping; spriteSheet.ts cache quantized to 0.5 increments; officeStore.ts stores raw float zoom; gameLoop.ts uses threshold for overview mode; click-to-walk targeting uses inverse transform math.
</objective>

  <name>Task 2: setTransform rendering + inverse-transform click targeting</name>

  <action>
Refactor renderer.ts to use ctx.setTransform() for all world-coordinate rendering, eliminating manual `* zoom` math and Math.floor tile positioning:

**renderer.ts — renderFrame():**
1. Compute world-space offsets (no zoom multiplication):
   ```
   const mapW = mapCols * TILE_SIZE;  // world pixels, not screen pixels
   const mapH = mapRows * TILE_SIZE;
   const offsetX = (canvasWidth / zoom - mapW) / 2 - camera.x / zoom;
   const offsetY = (canvasHeight / zoom - mapH) / 2 - camera.y / zoom;
   ```
   Wait — this is getting complex. Simpler approach matching the research:

   Compute screen-space offsets the same way as before BUT without Math.floor/Math.round:
   ```
   const tileSize = TILE_SIZE * zoom;
   const mapW = mapCols * tileSize;
   const mapH = mapRows * tileSize;
   const offsetX = (canvasWidth - mapW) / 2 - camera.x;
   const offsetY = (canvasHeight - mapH) / 2 - camera.y;
   ```

   Then apply the transform: `ctx.setTransform(zoom, 0, 0, zoom, offsetX, offsetY);`
   Wait — that's not right either. setTransform(zoom, 0, 0, zoom, tx, ty) means: scale by zoom, then translate by (tx, ty). So coordinates drawn at (col * TILE_SIZE, row * TILE_SIZE) in world space get rendered at (col * TILE_SIZE * zoom + tx, row * TILE_SIZE * zoom + ty). We want the map centered in canvas offset by camera. So:

   ```
   const mapWorldW = mapCols * TILE_SIZE;
   const mapWorldH = mapRows * TILE_SIZE;
   const tx = (canvasWidth - mapWorldW * zoom) / 2 - camera.x;
   const ty = (canvasHeight - mapWorldH * zoom) / 2 - camera.y;
   ctx.setTransform(zoom, 0, 0, zoom, tx, ty);
   ```

   Wait, setTransform applies translation AFTER scaling, so a point at world (wx, wy) goes to screen (wx * zoom + tx, wy * zoom + ty). If we want the map centered: the center of the map is at world (mapWorldW/2, mapWorldH/2). We want that to appear at screen (canvasWidth/2, canvasHeight/2) when camera is at (0,0). So:
   `canvasWidth/2 = (mapWorldW/2) * zoom + tx` → `tx = canvasWidth/2 - mapWorldW * zoom / 2 - camera.x`

   This simplifies to: `tx = (canvasWidth - mapWorldW * zoom) / 2 - camera.x`

   Yes, that's the same formula. So:
   ```typescript
   const mapWorldW = mapCols * TILE_SIZE;
   const mapWorldH = mapRows * TILE_SIZE;
   const tx = (canvasWidth - mapWorldW * zoom) / 2 - camera.x;
   const ty = (canvasHeight - mapWorldH * zoom) / 2 - camera.y;
   ```

2. After clearing canvas (Layer 1 — clear at identity transform first), apply the world transform:
   ```
   ctx.setTransform(1, 0, 0, 1, 0, 0);  // identity for clear
   ctx.fillStyle = BG_COLOR;
   ctx.fillRect(0, 0, canvasWidth, canvasHeight);
   ctx.setTransform(zoom, 0, 0, zoom, tx, ty);
   ctx.imageSmoothingEnabled = false;
   ```

3. **Floor tiles (Layer 2):** Draw at world coordinates — no manual zoom math:
   - Replace `Math.floor(col * tileSize + offsetX)` with just `col * TILE_SIZE` and `row * TILE_SIZE`
   - For sprite tiles: use `getCachedSprite(envSheet, frame, zoom)` — but now the cached sprite is pre-scaled to zoom, and we're drawing into a zoom-transformed context. This would double-zoom! Instead: draw the SOURCE sprite frame directly at world coords:
     `ctx.drawImage(sheet, frame.x, frame.y, frame.w, frame.h, col * TILE_SIZE, row * TILE_SIZE, TILE_SIZE, TILE_SIZE);`
     This lets the setTransform handle scaling. No pre-scaled cache needed for tiles drawn via setTransform. However, `imageSmoothingEnabled = false` with setTransform still does nearest-neighbor, so pixel art stays crisp.

   **IMPORTANT DECISION:** With setTransform, we have two options:
   (a) Draw source sprites at world coords and let transform scale them — simpler, no cache needed for world rendering
   (b) Keep getCachedSprite but account for zoom already in transform — complex, double-scaling risk

   **Use option (a):** Draw sprites from source sheet at world coordinates. Remove getCachedSprite usage for world-layer rendering (floor, furniture, characters). The setTransform handles zoom scaling. This eliminates the tile-gap problem entirely because canvas handles sub-pixel positioning uniformly.

   For UI overlays that render at screen coordinates (status bubbles, room labels, file icons), reset to identity transform and use screen-space math.

4. **Viewport culling:** Adjust to use world coords:
   ```
   const minCol = Math.max(0, Math.floor(-tx / zoom / TILE_SIZE));
   ```
   Actually simpler: compute visible world rect from screen bounds:
   ```
   const worldLeft = -tx / zoom;
   const worldTop = -ty / zoom;
   const worldRight = (canvasWidth - tx) / zoom;
   const worldBottom = (canvasHeight - ty) / zoom;
   const minCol = Math.max(0, Math.floor(worldLeft / TILE_SIZE));
   const maxCol = Math.min(mapCols - 1, Math.floor(worldRight / TILE_SIZE));
   const minRow = Math.max(0, Math.floor(worldTop / TILE_SIZE));
   const maxRow = Math.min(mapRows - 1, Math.floor(worldBottom / TILE_SIZE));
   ```

5. **Furniture (Layer 3):** Same approach — draw at world coords `item.col * TILE_SIZE, item.row * TILE_SIZE`. For sprite furniture, use `ctx.drawImage(sheet, frame.x, frame.y, frame.w, frame.h, worldX, worldY, w * TILE_SIZE, h * TILE_SIZE)`. For fallback colored rects, use `ctx.fillRect(worldX, worldY, item.width * TILE_SIZE, item.height * TILE_SIZE)`.

6. **Characters (Layer 4):** Draw at world coords. Character position `ch.x, ch.y` is already in world pixels. Draw character sprite at `(ch.x, ch.y)` — the transform handles zoom. Fallback colored rectangle: `ctx.fillRect(ch.x + pad, ch.y + pad, size, size)` where `size = TILE_SIZE * 0.8` and `pad = TILE_SIZE * 0.1` (world units, no zoom multiplication).

7. **Decorations:** Draw at world coords: `col * TILE_SIZE, row * TILE_SIZE`.

8. **Drop zone highlight (Layer 3b):** Draw at world coords using TILE_SIZE.

9. **UI Overlays (Layers 4b, 4c, 5, 5b):** Reset transform BEFORE drawing these:

   ```
   function worldToScreen(worldX: number, worldY: number): { x: number; y: number } {
     return { x: worldX * zoom + tx, y: worldY * zoom + ty };
   }
   ```

   Use this for all overlay positioning. The existing Math.floor calls in overlay code are fine for screen-space pixel snapping.

Actually, to minimize churn: keep function signatures but change the semantics — `offsetX` becomes `tx`, `tileSize` stays but equals `TILE_SIZE` (constant) for world-layer functions, or equals `TILE_SIZE * zoom` for screen-layer functions. Better: just change what's needed, keep working.
