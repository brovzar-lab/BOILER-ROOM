**AudioManager (src/engine/audioManager.ts):**
Create a singleton class accessed via `getAudioManager()`:

```typescript
class AudioManager {
  private audioContext: AudioContext | null = null;
  private ambientSource: AudioBufferSourceNode | null = null;
  private sfxBuffers: Map<string, AudioBuffer> = new Map();
  private ambientBuffer: AudioBuffer | null = null;
  private ambientGain: GainNode | null = null;
  private sfxGain: GainNode | null = null;
  private loadedSfx: Set<string> = new Set();
}
```

Key behaviors:
- `ensureContext()`: Creates AudioContext lazily on first user interaction (browser autoplay policy). Must be called from a click/keydown handler.
- `loadAmbient()`: Fetch and decode ambient-office.mp3 on first unmute. Do NOT eager-load.
- `loadSfx(name)`: Fetch and decode a specific SFX file on first trigger. Cache in sfxBuffers map.
- `playAmbient(roomId)`: Start looping ambient with volume adjustment per room:
  - Agent offices: ambientVolume * 0.6 (quieter)
  - War Room: ambientVolume * 1.0 (more energetic)
  - Hallway/Billy: ambientVolume * 0.8
  Only plays if !ambientMuted (reads audioStore.getState()).
- `stopAmbient()`: Stop ambient loop.
- `updateAmbientForRoom(roomId)`: Smooth volume crossfade (200ms) when room changes.
- `playSfx(name)`: Play a one-shot SFX if !sfxMuted. Lazy-loads the file on first trigger.
- `setAmbientMuted(muted)`: Start/stop ambient based on mute state.
- `setSfxMuted(muted)`: Just updates internal state (SFX are one-shot, no need to stop).

SFX trigger points (documented, wired in Task 2):
- 'footstep': Every ~0.3s during BILLY walk state
- 'knock': When knockTimer is first set in characters.ts
- 'paper': When fileStore.addFile is called
- 'chime': When agentStatus transitions from 'thinking' to 'idle' or 'needs-attention'
