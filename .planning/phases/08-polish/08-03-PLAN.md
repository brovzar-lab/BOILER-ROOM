**camera.ts changes:**
- Add a `computeAutoFitZoom(canvasWidth: number, canvasHeight: number): number` function:
  - Calculate the zoom level that fits the entire 42x34 tile map (42*16=672px wide, 34*16=544px tall) within the canvas dimensions
  - Formula: `Math.max(1, Math.floor(Math.min(canvasWidth / (42 * TILE_SIZE), canvasHeight / (34 * TILE_SIZE))))`
  - Returns an integer zoom level (1, 2, 3, etc.) -- maintains pixel-perfect rendering
  - This gives the largest integer zoom where the full map fits
- Export `computeAutoFitZoom`
- Update `createCamera()`: keep zoom=2 as default (auto-fit is applied after canvas mounts)

**gameLoop.ts changes:**
- On first frame (lastTime === 0), call computeAutoFitZoom(canvasWidth, canvasHeight) and set camera.zoom to the result. Also update state.setZoom if officeStore has such a setter, or directly mutate camera.zoom.
- On canvas resize (when prevWidth/prevHeight change), if camera.zoom was auto-fit (add a flag `let isAutoFitZoom = true`), recalculate and apply auto-fit zoom. Once the user manually zooms, set `isAutoFitZoom = false` and stop auto-adjusting.

**ZoomControls.tsx changes:**
- Add an "auto-fit" button (or make it a third option alongside +/-): a small icon button that resets zoom to auto-fit value. Use a "fit" icon (arrows pointing inward).
- When + or - is clicked, mark that the user has manually overridden zoom (communicate this to gameLoop via officeStore or a simple flag on camera).
- Support zoom levels: 1 through computeAutoFitZoom max (not just 1 and 2). At 1920px width with ~1200px canvas, auto-fit might give zoom=3.
- Keep follow mode logic: zoom >= 2 follows BILLY, zoom === 1 is overview.

**officeStore.ts changes (minimal):**
- If setZoom action doesn't exist, add one that updates camera.zoom on the characters state. Check existing store shape first.

**App.tsx layout changes:**

This is a CHANGE from the current layout per CONTEXT.md: "Side-by-side layout: canvas and chat panel sit next to each other" and "At narrow widths (near 1280px): chat collapses to an icon and can be expanded."

Current layout:
```
[DealSidebar] [Canvas (flex-1 relative)] [ChatPanel (w-[400px] shrink-0)]
```

New layout:
```
[DealSidebar] [Canvas (flex-1 min-w-0)] [ChatPanel (w-[400px] max-w-[500px] collapsible)]
```

Changes:
1. Add `chatOpen` state (default true) alongside existing `sidebarOpen`
2. Chat panel column:
   - When `chatOpen`: `w-[400px] max-w-[500px] shrink-0` with existing ChatPanel + FileViewer content
   - When `!chatOpen`: render a small floating button (fixed right edge) with a chat bubble icon to re-open
3. Add responsive breakpoint detection:
   - Use a ResizeObserver on the main container (or window.innerWidth)
   - When width < 1400px: auto-collapse chat panel (set chatOpen=false), show expand button
   - When width >= 1400px: auto-expand chat panel (set chatOpen=true)
   - User can always manually toggle -- manual toggle overrides auto-behavior until next resize crosses threshold
4. Chat collapse button: Add a small close/collapse button in the chat panel's top-right corner (next to existing content)
5. Chat expand button: When collapsed, show a floating rounded button at the right edge of the canvas area with a chat bubble icon + agent name badge
6. Canvas container: keep `flex-1 relative overflow-hidden min-w-0` -- the min-w-0 ensures it shrinks properly in flex
7. At ultrawide (container > 2000px): chat panel stays at max-w-[500px], canvas gets all remaining space

**OfficeCanvas.tsx changes:**
- The ResizeObserver already handles canvas resize correctly
- When chat panel collapses/expands, canvas will auto-resize via flex and ResizeObserver catches it
- No code changes needed unless the resize triggers need debouncing (currently they don't)

**Key responsive behaviors:**
- 1280px total: sidebar closed by default, chat collapsed, canvas gets full width
- 1440px: sidebar + canvas + chat all visible
- 1920px: comfortable spacing, auto-fit zoom likely gives zoom=2
- 2560px: canvas fills extra space, chat at max 500px
- Transitions between states should be smooth (use Tailwind transition-all duration-200)
