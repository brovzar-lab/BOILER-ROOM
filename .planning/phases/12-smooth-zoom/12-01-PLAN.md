  <files>src/engine/zoomController.ts, src/engine/camera.ts, src/engine/__tests__/zoomController.test.ts</files>
  <behavior>
    - Test: exponential zoom factor produces correct new zoom from delta (e.g., delta=-100 with ZOOM_FACTOR=1.002 produces zoom * 1.002^100)
    - Test: cursor-centered zoom keeps world point invariant (world coords of cursor before and after zoom are identical)
    - Test: inertia decays velocity toward zero over multiple ticks
    - Test: after inertia stops and idle delay (150ms) passes, snap target is nearest 0.5
    - Test: snap animation eases from current zoom to snap target over SNAP_DURATION
    - Test: skip snap if zoom already within 0.05 of a half-integer
    - Test: zoom is clamped between dynamic minZoom and MAX_ZOOM (4.0)
    - Test: pixel-align camera (round x/y to integers) after snap completes
  </behavior>
  <action>
    Create `src/engine/zoomController.ts` with:

    1. **ZoomAnimState interface** with fields: velocity (log-space), idleTimer, snapTarget, snapProgress, snapStartZoom, phase ('idle' | 'input' | 'inertia' | 'settling' | 'snapping'), lastCursorX, lastCursorY.

    2. **Constants:** ZOOM_FACTOR = 1.002 (per pixel of deltaY), FRICTION = 0.92 (per-frame velocity decay), IDLE_DELAY = 0.15s, SNAP_DURATION = 0.25s, SNAP_THRESHOLD = 0.05, MAX_ZOOM = 4.0, MIN_VELOCITY = 0.001.

    3. **createZoomState()** factory returning initial idle state.

    4. **onZoomInput(state, rawDelta, cursorScreenX, cursorScreenY)** -- called from wheel handler. Converts rawDelta to velocity, sets phase to 'input', stores cursor position. Does NOT directly change camera.zoom (that happens in tickZoom).

    5. **tickZoom(state, camera, dt, canvasWidth, canvasHeight, minZoom)** -- called each frame from game loop. State machine transitions:
       - 'input' -> 'inertia': when no input received this frame (check a "inputThisFrame" flag)
       - 'inertia': apply velocity to camera.zoom via exponential scaling, apply FRICTION to velocity per frame. Call applyCursorCenteredZoom. When velocity < MIN_VELOCITY -> 'settling'.
       - 'settling': increment idleTimer. When >= IDLE_DELAY -> compute snapTarget = nearestHalf(camera.zoom). If within SNAP_THRESHOLD -> snap instantly + pixel-align -> 'idle'. Else -> 'snapping'.
       - 'snapping': increment snapProgress by dt/SNAP_DURATION. Ease-out: `1 - Math.pow(1 - t, 3)`. Interpolate from snapStartZoom to snapTarget. Call applyCursorCenteredZoom with last cursor position. When progress >= 1 -> set exact snap target + pixel-align -> 'idle'.
       - 'idle': no-op.
       Clamp camera.zoom to [minZoom, MAX_ZOOM] after every change.

    6. **startAnimatedZoom(state, targetZoom, cursorScreenX, cursorScreenY)** -- for button clicks. Sets snapTarget, snapStartZoom, snapProgress=0, phase='snapping', stores cursor.

    7. **nearestHalf(zoom)** and **pixelAlignCamera(camera)** as exported helpers.

    Update `src/engine/camera.ts`:
    - **applyCursorCenteredZoom(camera, oldZoom, newZoom, cursorScreenX, cursorScreenY, canvasWidth, canvasHeight)** -- new exported function. Uses the formula from research: compute world coords of cursor at oldZoom, compute new camera.x/y so same world point maps to same screen point. Set both camera.x = camera.targetX and camera.y = camera.targetY to prevent lerp fighting.
    - **Remove `Math.max(1, ...)` clamp** in computeAutoFitZoom -- return raw `Math.min(canvasWidth / mapPixelW, canvasHeight / mapPixelH)` so auto-fit can go below 1.0 on large screens. Add a `Math.max(0.5, ...)` floor to prevent absurdly small zoom.

    Write tests in `src/engine/__tests__/zoomController.test.ts` covering all behavior items above. Mock camera as a plain object. Test tickZoom by calling it repeatedly with dt increments and asserting state transitions and camera.zoom values.
  </action>
