  <action>
    **gameLoop.ts changes:**

    1. Import `tickZoom` from `./zoomController`, `zoomState` and `cursorScreenX`/`cursorScreenY` from `./input`.

    2. Add a `prevQuantizedZoom` variable (initialized to 0) alongside other tracking state.

    3. In the frame() function, AFTER updateAllCharacters and BEFORE the camera follow/target logic, add zoom tick:
       ```
       const minZoom = computeAutoFitZoom(canvasWidth, canvasHeight);
       tickZoom(zoomState, state.camera, dt, canvasWidth, canvasHeight, minZoom);
       ```

    4. After tickZoom, sync store ONLY when quantized zoom changes (prevents React re-render spam):
       ```
       const quantized = Math.round(state.camera.zoom * 2) / 2;
       if (quantized !== prevQuantizedZoom) {
         prevQuantizedZoom = quantized;
         useOfficeStore.getState().setZoomLevel(state.camera.zoom);
       }
       ```

    5. **Remove the old manual zoom override detection block** (`if (state.zoomLevel !== state.camera.zoom)`). The zoom controller now owns all zoom changes -- the store is written TO by the game loop, not read FROM. Camera.zoom is the source of truth during animation; store is synced for React.

    6. **Update the auto-fit logic**: The `__boiler_reset_autofit` global flag should now also call `startAnimatedZoom(zoomState, fitZoom, canvasWidth/2, canvasHeight/2)` instead of setting camera.zoom directly, so it animates. Keep the `isAutoFitZoom = true` reset.

    7. **Update resize auto-fit**: When `isAutoFitZoom && resize detected`, use `startAnimatedZoom` for smooth resize transitions (or set directly if it's the first frame -- no point animating load).

    8. **Camera follow reconciliation with active zoom** (per user decision: camera keeps following BILLY during zoom -- no pause of follow mode): Instead of suppressing the follow system when zoom is active, apply the cursor-centered zoom offset as a DELTA on top of the follow target. The approach:
       - Let the follow system compute targetX/targetY for BILLY as normal every frame, regardless of zoom phase.
       - In tickZoom (or immediately after), the cursor-centered zoom math (applyCursorCenteredZoom) adjusts camera.x and camera.y. These adjustments shift the camera relative to the follow target.
       - The follow lerp continues pulling camera toward BILLY's position, while the zoom math continuously re-centers on the cursor. The result is a blend: the camera zooms around the cursor point but also drifts toward BILLY between zoom inputs.
       - When zoom becomes 'idle', the follow lerp naturally takes full control again since no more zoom offsets are applied.
       - This means: do NOT skip or gate the BILLY follow logic based on zoomState.phase. The follow system runs unconditionally. The zoom controller's applyCursorCenteredZoom runs on top of whatever camera position the follow system set.

    **ZoomControls.tsx changes:**

    1. Import `startAnimatedZoom`, `nearestHalf`, `MAX_ZOOM` from `@/engine/zoomController`, `zoomState` from `@/engine/input`, `computeAutoFitZoom` from `@/engine/camera`.

    2. **Remove mode label** entirely (no more "Overview"/"Follow").

    3. **Remove auto-fit button** entirely (0 key handles this per user decision).

    4. **handleZoomIn**: compute `target = nearestHalf(zoomLevel) + 0.5`, clamp to MAX_ZOOM. Call `startAnimatedZoom(zoomState, target, window.innerWidth / 2, window.innerHeight / 2)`. Also call `setZoomLevel(target)` so React state updates for the disabled check.

    5. **handleZoomOut**: compute `target = nearestHalf(zoomLevel) - 0.5`, clamp to `computeAutoFitZoom(window.innerWidth, window.innerHeight)`. Call `startAnimatedZoom(zoomState, target, window.innerWidth / 2, window.innerHeight / 2)`. Also call `setZoomLevel(target)`.

    6. **Styling updates** per user decisions:
       - Buttons: `w-6 h-6` (24px from 32px), `opacity-40 hover:opacity-100` transition
       - Remove the gap between mode label area (deleted) and buttons
       - Add `title` attributes for tooltips: "Zoom in (+/=)" and "Zoom out (-)"
       - Keep `bottom-6 right-6` positioning
       - Keep `bg-[--color-surface-card]/80` background but apply it at lower opacity by default

    7. **Disable logic**: disabled when zoom at max (for +) or at auto-fit min (for -). Use `nearestHalf(zoomLevel) + 0.5 > MAX_ZOOM` for + button, `nearestHalf(zoomLevel) - 0.5 < minZoom` for - button.
