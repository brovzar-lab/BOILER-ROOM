<objective>
Build the War Room UI: useWarRoom orchestration hook, WarRoomPanel with multi-stream display, ChatPanel routing, War Room badge, and cross-visibility context injection.

Purpose: This is the integration plan that wires Plans 01 (store/types) and 02 (engine/animations) into a working War Room experience. The user enters the War Room, agents gather at the table, user types a message, 5 parallel streams fire simultaneously, responses display color-coded and labeled in the panel, and each response is mirrored back to individual agent conversation histories. Partial failures are handled gracefully (WAR-05), and rate limits trigger auto-retry (WAR-06). A human verification checkpoint confirms the full flow works visually.

Output: Complete War Room feature with useWarRoom hook, WarRoomPanel, WarRoomMessage, WarRoomBadge, updated ChatPanel and MessageBubble, updated context builder.

    - Track gathering state: use a local `isGathering` state that flips to false when `gatherAgentsToWarRoom` promise resolves
    - Return `{ sendBroadcast, cancelAll, isGathering, warRoomStreaming, warRoomRound }`
    - Cleanup: on unmount, if any streams active, cancel them
