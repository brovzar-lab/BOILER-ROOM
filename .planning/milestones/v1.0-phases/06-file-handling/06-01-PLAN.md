First install dependencies: `npm install pdfjs-dist mammoth`

Create `src/types/file.ts` with FileRecord interface:
```typescript
export interface FileRecord {
  id: string;          // crypto.randomUUID()
  name: string;        // Original filename
  size: number;        // Original file size in bytes
  type: 'pdf' | 'docx';
  agentId: string;     // Which agent's desk
  dealId: string;      // Which deal
  extractedText: string;
  uploadedAt: number;  // Date.now()
}
```

Create `src/services/files/extractPdf.ts`:
- Import pdfjs-dist and configure workerSrc using Vite `?url` import: `import pdfjsWorker from 'pdfjs-dist/build/pdf.worker.min.mjs?url'`
- Set `pdfjsLib.GlobalWorkerOptions.workerSrc = pdfjsWorker`
- Export `extractPdfText(data: ArrayBuffer): Promise<string>` that uses `getDocument({data}).promise`, iterates pages with `getPage(i)`, calls `getTextContent()`, joins item.str values with spaces, joins pages with double newlines

Create `src/services/files/extractDocx.ts`:
- Import mammoth
- Export `extractDocxText(data: ArrayBuffer): Promise<string>` that calls `mammoth.extractRawText({ arrayBuffer: data })` and returns `result.value`

Create `src/services/files/fileService.ts`:
- Import extractPdfText, extractDocxText, getPersistence, AgentId, FileRecord
- `processDroppedFile(file: File, agentId: AgentId, dealId: string): Promise<FileRecord>` -- reads file.arrayBuffer(), detects type by extension (.pdf/.docx), calls appropriate extractor, creates FileRecord with crypto.randomUUID(), persists via getPersistence().set('files', record.id, record), returns record. Throws Error for unsupported extensions.
- `deleteFile(fileId: string): Promise<void>` -- calls getPersistence().delete('files', fileId)
- `getFilesForAgent(agentId: string, dealId: string): Promise<FileRecord[]>` -- queries by agentId index, filters results by dealId
- `shareFileWithAllAgents(fileRecord: FileRecord, dealId: string): Promise<FileRecord[]>` -- for each of the 5 AgentIds, creates a copy with new UUID, same extractedText, persists each, returns array of new records

Mock pdfjs-dist and mammoth in tests. Mock getPersistence() to use an in-memory store. Test the orchestration logic: correct type detection, correct FileRecord shape, persistence calls, error on unsupported types.

Replace the stub in `src/store/fileStore.ts` with a full Zustand store following the established pattern (flat state + actions):

```typescript
interface FileState {
  files: FileRecord[];
  isProcessing: boolean;

  addFile: (file: File, agentId: AgentId) => Promise<void>;
  removeFile: (fileId: string) => Promise<void>;
  loadFiles: (agentId: AgentId, dealId: string) => Promise<void>;
  shareFile: (fileRecord: FileRecord) => Promise<void>;
  getFilesByAgent: (agentId: AgentId) => FileRecord[];
}
```

- `addFile`: reads `useDealStore.getState().activeDealId` for the current deal (default to 'default'), sets isProcessing=true, calls `processDroppedFile(file, agentId, dealId)`, appends result to files array, sets isProcessing=false. Wrap in try/catch, reset isProcessing on error.
- `removeFile`: calls `deleteFile(fileId)`, filters out from files array
- `loadFiles`: calls `getFilesForAgent(agentId, dealId)`, sets files to result
- `shareFile`: calls `shareFileWithAllAgents(fileRecord, dealId)`, concatenates new records onto files
- `getFilesByAgent`: pure filter on files array (not async, uses get())

Tests should mock fileService functions and dealStore. Verify state transitions (isProcessing toggling), files array mutations, error handling.
