**FileViewer.tsx:**

Create a slide-out panel component (same position/style as ChatPanel -- right side, 400px wide) that overlays the chat panel with a higher z-index per CONTEXT.md decisions.

Props: `fileId: string | null, onClose: () => void`

When fileId is non-null:
- Read the FileRecord from fileStore by id
- Header section: file name (bold), file size (formatted as KB/MB), upload date (formatted with toLocaleDateString), agent name (look up from agent registry via agentId), file type badge (PDF red / DOCX blue)
- Body: extracted text rendered with whitespace preserved (pre-wrap). Use basic markdown-like formatting if possible, or plain text with good typography
- Footer: Close button (styled consistently with app theme) and Delete button (red accent). Per user decision: close and delete/remove buttons ONLY -- no copy, no re-extract, no share button in this panel.
- Delete button calls `useFileStore.getState().removeFile(fileId)` then calls `onClose()`

When fileId is null, render nothing.

Animation: slide in from right (translate-x transition, 200ms).

Styling: dark theme consistent with app (bg-neutral-900, text-neutral-100, amber accents for buttons).

NOTE: The "Share with all agents" action is available via the fileStore.shareFile() function but is NOT exposed in the FileViewer per locked user decision. It can be surfaced in a future file list dropdown or context menu if needed.

**FileViewer tests (src/components/__tests__/FileViewer.test.tsx):**
- Test that FileViewer renders file metadata (name, size, date, agent name) when fileId matches a file in the store
- Test that clicking Delete calls removeFile(fileId) and then onClose
- Test that FileViewer renders nothing when fileId is null
- Test that only Close and Delete buttons are present (no Share button)
- Mock useFileStore with a test FileRecord

**ChatPanel drop zone:**

Modify ChatPanel (the outer component or AgentChatPanel inner) to accept file drops:
- Add `onDragOver` handler that calls preventDefault and sets a visual state (e.g., a border highlight around the panel)
- Add `onDrop` handler that extracts files from dataTransfer, calls `useFileStore.getState().addFile(file, activeAgentId)` for each .pdf/.docx file
- Add `onDragLeave` handler that removes the visual highlight
- Visual feedback: thin amber border + "Drop file here" text overlay when dragging over the panel
- Only show drop zone when an agent room is active (not War Room, not BILLY's office, not overview)

**Header file count indicator:**

Modify Header component to show a file count badge when the active agent has files:
- Read files from fileStore filtered by active agentId and active dealId
- If count > 0, show a small icon (document emoji or a styled span) + count (e.g., "3 files") next to the agent name
- Clicking the indicator could open a file list dropdown (or simply be informational for now)

**Canvas file click -> FileViewer wiring:**

In App.tsx (or OfficeCanvas component):
- Track `selectedFileId: string | null` in local state
- On mount (or when canvas is set up), set `onFileClickCallback` from input.ts to `(fileId) => setSelectedFileId(fileId)`. This uses the callback export from Plan 02 -- no modifications to input.ts needed.
- Render `<FileViewer fileId={selectedFileId} onClose={() => setSelectedFileId(null)} />` positioned above the chat panel (z-50 vs chat panel's z-40).

**Processing indicator:**

When fileStore.isProcessing is true, show a small pulsing indicator on the active agent's desk area or in the chat panel header -- a subtle "Processing file..." text or a pulsing document icon. Per CONTEXT.md: "Document icon on the desk pulses/animates until extraction completes. No toast notification."
