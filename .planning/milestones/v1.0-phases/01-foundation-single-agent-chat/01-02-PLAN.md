2. **src/services/anthropic/client.ts**: Create a configured Anthropic SDK client instance using `@anthropic-ai/sdk` per user decision (the SDK supports browser environments via `dangerouslyAllowBrowser: true` since it uses standard `fetch` internally):
   ```typescript
   import Anthropic from '@anthropic-ai/sdk';

   let clientInstance: Anthropic | null = null;

   export function getAnthropicClient(): Anthropic {
     if (!clientInstance) {
       clientInstance = new Anthropic({
         // API key is injected by the Vite proxy, so we pass a dummy value here.
         // The SDK requires apiKey to be set, but the proxy overwrites the header.
         apiKey: 'proxy-handled',
         baseURL: '/api/anthropic',
         dangerouslyAllowBrowser: true,
       });
     }
     return clientInstance;
   }

   // Custom error wrapper for retry logic
   export class AnthropicError extends Error {
     constructor(public status: number, message: string) {
       super(message);
       this.name = 'AnthropicError';
     }
     get isRetryable(): boolean {
       return this.status === 429 || this.status === 500 || this.status === 529;
     }
   }
   ```
   Key design: The SDK sends requests to `baseURL` (our Vite proxy at `/api/anthropic`). The proxy rewrites the path and injects the real API key server-side. The client bundle never contains the real API key. The `dangerouslyAllowBrowser` flag is safe here because we are NOT exposing the real key -- it is proxy-handled.
